# ComfyUI Asset Manager

ComfyUI 서버에 연결하여 대량의 이미지를 모듈화된 프롬프트로 생성·관리하는 데스크탑 애플리케이션입니다.

![Vue 3](https://img.shields.io/badge/Vue-3.5-4FC08D?logo=vuedotjs)
![TypeScript](https://img.shields.io/badge/TypeScript-5.9-3178C6?logo=typescript)

## 주요 기능

> **v0.7.1**: MCP 서버 멀티 CLI 호환 (Copilot CLI, Claude Code, Gemini CLI, Codex CLI). 세션 관리 버그 수정.

> **v0.6.0**: UI 리디자인으로 7페이지 → 4+1 구조로 간소화. 연결 상태는 상단 헤더 바에 표시됩니다.

### 🔗 ComfyUI 연결
- 로컬 ComfyUI Portable 서버에 REST + WebSocket으로 연결
- 상단 헤더 바에 실시간 연결 상태 표시 및 자동 재연결
- 시스템 통계 (GPU, VRAM) 조회

### 📋 워크플로우 관리
- ComfyUI API 형식 JSON 워크플로우 가져오기
- 자동 변수 감지 (시드, 프롬프트, 모델, LoRA 등)
- **프롬프트 슬롯 자동 감지**: CLIPTextEncode → KSampler 연결 추적으로 긍정/부정 역할 자동 판별
- 카테고리 자동 분류: 생성 / 업스케일 / 디테일러 / 커스텀
- 변수 역할 수동 편집: 긍정 프롬프트 / 부정 프롬프트 / 시드 / 고정값 / 사용자 정의

### 🧩 프롬프트 모듈 시스템
- 9가지 모듈 타입: 캐릭터, 복장, 감정, 스타일, 아티스트, 품질, 네거티브, LoRA, 커스텀
- 카드 그리드 레이아웃, 필 스타일 필터 버튼
- 모듈별 아이템 관리 (프롬프트, 네거티브, 가중치)
- **드래그 앤 드롭**으로 아이템 순서 변경
- 와일드카드 지원: `{red|blue|green}` → 랜덤 선택
- 변수 보간: `{{character_name}}` → 값 치환
- 실시간 프롬프트 미리보기

### 🔄 작업 (배치 + 큐 통합)
- **통합 작업 페이지**: 실행 상태 바 + 작업 카드 그리드로 배치 생성과 큐 관리를 한 화면에서 처리
- **3단계 배치 위자드**: 기본 설정 → 모듈 & 프롬프트 → 확인 (기존 7섹션 모달 대체)
- **매트릭스 빌더**: 모듈 조합의 카르테시안 곱으로 대량 작업 생성
  - 예: 캐릭터 3명 × 복장 5벌 × 감정 20개 × 조합당 100장 = 30,000장
- **프롬프트 슬롯 매핑 (간소화)**: 고정 모듈 + 조합 모듈 + 추가 텍스트 3개 입력으로 구성
  - 고정 모듈은 **드래그 앤 드롭**으로 순서 변경 가능 (프롬프트 합성 순서에 직접 영향)
  - 멀티 모델 워크플로우에서 어떤 슬롯에 모듈 프롬프트를 주입할지 명시적 설정
  - **슬롯별 모듈 할당**: 각 슬롯에 매트릭스의 어떤 모듈을 주입할지 체크박스로 선택
  - **고정 모듈 프리픽스**: 품질·스타일·아티스트 등 기존 모듈을 프리픽스로 다중 선택 (자동 합성)
- 순차 작업 처리 (ComfyUI에 프롬프트 제출 → 결과 대기 → 이미지 저장)
- 일시정지 / 재개 / 취소 지원
- 자동 재시도 (설정 가능한 최대 횟수)
- 실시간 진행률 표시
- **배치 작업 수정 / 재실행 / 복제**
- 시드 모드: 랜덤 / 고정 / 증분
- 출력 폴더·파일명 패턴 설정 (`{character}/{outfit}/{emotion}` 등)

### 🖼️ 갤러리

- 출력 디렉토리 및 폴더/파일명 패턴
- 다크/라이트 테마
- 한국어/영어 UI
- MCP 서버 활성화/포트 설정

### 🤖 MCP 서버 + 내장 터미널
- **MCP (Model Context Protocol) 서버**: LLM CLI(Copilot CLI, Claude CLI, Gemini CLI, Codex CLI)가 앱 기능을 도구로 사용 가능
  - 15개 핵심 도구: 모듈 CRUD, 아이템 CRUD, 워크플로우 조회, 배치 작업 생성/실행
  - Streamable HTTP 전송 방식 (최신 MCP 표준)
  - 설정에서 활성화/비활성화, 포트 설정
  - **멀티 CLI 자동 설정**: `.mcp.json` (Copilot/Claude), `~/.gemini/settings.json` (Gemini), `~/.codex/config.toml` (Codex)

  - xterm.js 기반 풀 PTY 터미널 (node-pty)
  - 멀티 탭 지원
  - **전용 페이지** + **하단 패널** 모드 전환 (VS Code 스타일)
  - 드래그로 패널 높이 조절

## 시작하기

### 필수 조건

- [Node.js](https://nodejs.org/) 18 이상
- [ComfyUI](https://github.com/comfyanonymous/ComfyUI) Portable 서버가 실행 중이어야 합니다 (기본: `localhost:8188`)

### 설치

```bash
git clone <repository-url>
cd comfyui_asset_manager
npm install
```

### 개발 모드

```bash
npm run dev
```

Electron 앱이 실행되며 렌더러에 HMR이 적용됩니다.

### 빌드

```bash
# 타입 체크 + 빌드
npm run build

# 플랫폼별 배포 패키지 생성
npm run build:win     # Windows
npm run build:mac     # macOS
npm run build:linux   # Linux
```

## 핵심 개념

이 프로그램을 효과적으로 사용하려면 세 가지 핵심 개념을 이해해야 합니다.

### 모듈과 아이템 — 조합의 차원

**모듈**은 프롬프트의 한 **차원(축)**이고, 모듈 안의 **아이템**은 그 축의 **값들**입니다.

| 모듈 (차원) | 아이템 (값) |
|------------|------------|
| 캐릭터 모듈 | 앨리스, 밥, 찰리 (3개) |
| 감정 모듈 | happy, sad, angry, surprised (4개) |
| 복장 모듈 | 교복, 드레스 (2개) |

배치 작업에서 이 모듈들을 선택하면 **카르테시안 곱(모든 조합)**이 자동 생성됩니다:

```
3 캐릭터 × 4 감정 × 2 복장 = 24가지 조합
× 조합당 100장 = 총 2,400장 자동 생성
```

### 고정 모듈 vs 조합 모듈

배치 마법사에서 프롬프트 슬롯에 모듈을 배치할 때, 두 가지 종류가 있습니다:

| 구분 | 고정 모듈 | 조합 모듈 |
|------|----------|----------|
| 역할 | 모든 조합에 **동일하게** 들어감 | 아이템별로 **달라지며** 매트릭스를 구성 |
| 용도 | 품질 태그, 작가 태그, 스타일 등 | 캐릭터, 감정, 복장 등 |
| 예시 | `masterpiece, best quality` | 앨리스 / 밥 / 찰리 |

**최종 프롬프트 구성 순서:**

```
[고정 모듈 프롬프트] + [추가 텍스트] + [조합 모듈 아이템 프롬프트] + [서픽스]
```

### 프롬프트 슬롯 — 어디에 넣을지 결정

복잡한 워크플로우(예: Anima 모델로 1차 생성 → ILXL 모델로 2차 i2i)는 CLIPTextEncode 노드가 4개 이상일 수 있습니다. **프롬프트 슬롯**은 "어떤 노드에 어떤 모듈의 프롬프트를 넣을지"를 명시적으로 설정하는 시스템입니다.

```
워크플로우 예시: Anima + ILXL 2단계 생성

┌─ Anima 긍정 프롬프트 슬롯 ───────────────────────┐
│ 고정 모듈: [아니마 작가태그], [품질태그]             │ ← 항상 동일
│ 추가 텍스트: "1girl, ..."                          │ ← 직접 입력
│ 조합 모듈: ☑ 캐릭터  ☑ 감정  ☑ 복장                │ ← 조합마다 변경
└──────────────────────────────────────────────────┘

┌─ Anima 부정 프롬프트 슬롯 ───────────────────────┐
│ 고정 모듈: [아니마 네거티브]                        │
│ 조합 모듈: (없음)                                  │
└──────────────────────────────────────────────────┘

┌─ ILXL 긍정 프롬프트 슬롯 ────────────────────────┐
│ 고정 모듈: [ILXL 작가태그]                         │
│ 조합 모듈: ☑ 캐릭터  □ 감정  ☑ 복장                │ ← 다르게 선택 가능!
└──────────────────────────────────────────────────┘

┌─ ILXL 부정 프롬프트 슬롯 ────────────────────────┐
│ 고정값: "worst quality, low quality"               │ ← 고정 텍스트 직접 입력
└──────────────────────────────────────────────────┘
```

이처럼 **슬롯마다 다른 모듈을 할당**할 수 있어, 한 워크플로우 안의 여러 모델에 각각 다른 프롬프트를 넣을 수 있습니다.

## 사용법

### 1. ComfyUI 연결

1. ComfyUI Portable 서버를 실행합니다
2. 앱 실행 시 자동 연결을 시도합니다 (기본: `localhost:8188`)
3. 상단 헤더 바의 연결 상태 버튼으로 수동 연결/해제할 수 있습니다
4. 연결 실패 시 **설정** 페이지에서 호스트·포트를 변경합니다

### 2. 워크플로우 등록

1. **워크플로우** 페이지 → **워크플로우 가져오기** 클릭
2. ComfyUI에서 **Save (API Format)** 으로 저장한 JSON 파일을 선택합니다
3. 자동으로 변수가 감지되고 카테고리가 분류됩니다
4. **프롬프트 슬롯이 자동 감지**됩니다 — 각 CLIPTextEncode 노드가 KSampler에 어떻게 연결되었는지 추적하여 긍정/부정 역할을 판별합니다
5. 필요시 워크플로우 **상세** 모달에서 변수의 역할을 수동으로 변경합니다

> ⚠️ UI 형식이 아닌 **API 형식** JSON만 지원합니다. ComfyUI에서 `Save (API Format)` 버튼으로 저장하세요.
>
> 💡 멀티 모델 워크플로우(예: Anima + ILXL 2단계)는 각 모델의 프롬프트 슬롯이 별도로 감지됩니다. 커스텀 노드를 사용하는 경우에도 역할을 수동으로 지정할 수 있습니다.

### 3. 프롬프트 모듈 만들기

1. **프롬프트 모듈** 페이지 → **새 모듈** 클릭
2. 모듈 이름, 타입(캐릭터/복장/감정 등)을 설정합니다
3. 카드를 선택하고 **아이템 추가**로 프롬프트 아이템을 등록합니다
4. 각 아이템에 프롬프트, 네거티브 프롬프트, 가중치를 설정합니다

**모듈 구성 예시:**

```
📁 캐릭터 모듈 "메인 캐릭터"
   ├── 아이템: "앨리스"  →  프롬프트: "alice, blue eyes, long blonde hair"
   ├── 아이템: "밥"      →  프롬프트: "bob, brown eyes, short black hair"
   └── 아이템: "찰리"    →  프롬프트: "charlie, green eyes, red twintails"

📁 감정 모듈 "기본 감정"
   ├── 아이템: "행복"    →  프롬프트: "happy, smile, bright expression"
   ├── 아이템: "슬픔"    →  프롬프트: "sad, crying, tears"
   └── 아이템: "분노"    →  프롬프트: "angry, frown, clenched teeth"

📁 품질 모듈 "아니마 품질태그"  (← 고정 모듈로 사용)
   └── 아이템: "기본"    →  프롬프트: "masterpiece, best quality, absurdres"
```

> 💡 **고정 모듈로 쓸 모듈**도 아이템이 최소 1개 필요합니다. 품질 태그나 작가 태그처럼 항상 동일하게 넣을 모듈은 아이템 1개만 만들면 됩니다.

**프롬프트 특수 문법:**
```
{red hair|blonde hair|silver hair}    → 와일드카드: 랜덤 선택
{{character_name}}                     → 변수 보간: 값 치환
```

### 4. 배치 작업 생성 & 실행

**작업** 페이지 → **새 배치 작업**을 클릭하면 3단계 위자드가 열립니다:

**1단계 — 기본 설정:**
- 작업 이름, 워크플로우 선택
- 조합당 생성 수 (예: 100)
- 시드 모드: 랜덤 / 고정 / 증분

**2단계 — 모듈 & 프롬프트** (가장 중요한 단계):

- **왼쪽: 조합 모듈 매트릭스**
  - 드롭다운에서 모듈을 추가하고 아이템을 체크박스로 선택합니다
  - 선택된 모듈 × 아이템의 카르테시안 곱이 전체 조합이 됩니다

- **오른쪽: 프롬프트 슬롯 매핑**
  - 워크플로우에서 감지된 각 프롬프트 슬롯이 나타납니다
  - 슬롯마다 설정할 수 있는 항목:
    - **고정 모듈**: 품질·작가 태그 등 매번 동일하게 들어갈 모듈을 다중 선택
    - **추가 텍스트**: `1girl, ...` 등 직접 입력하는 고정 텍스트
    - **조합 모듈**: 매트릭스의 어떤 모듈을 이 슬롯에 주입할지 체크박스로 선택
    - **서픽스**: 뒤에 붙일 추가 텍스트

**3단계 — 확인 & 생성:**
- 총 조합 수와 이미지 수 미리보기
- 변수 오버라이드 (선택사항): 모델, LoRA 가중치, 샘플러 등 변경
- 출력 폴더/파일명 패턴 설정
- **생성** 버튼 클릭

**실행:**
- 작업 카드의 **시작** 버튼을 클릭하면 실행됩니다
- 상단 상태 바에서 실시간 진행률과 일시정지/재개/취소 가능
- 완료된 작업은 **수정/재실행/복제** 가능

### 5. 결과 확인

1. **갤러리** 페이지에서 생성된 이미지를 확인합니다
2. 캐릭터·감정별 필터, ⭐ 평점, ♥ 즐겨찾기로 관리합니다
3. 출력 폴더에서 `{작업명}/{캐릭터}/{복장}/{감정}/` 구조로 파일을 직접 확인할 수도 있습니다

### 전체 워크플로우 예시

> Anima + ILXL 2단계 워크플로우로 캐릭터 3명 × 복장 2벌 × 감정 4개, 조합당 50장 생성하기

1. ComfyUI에서 Anima → ILXL 워크플로우를 API 형식으로 저장
2. 워크플로우 가져오기 → 4개의 프롬프트 슬롯이 자동 감지됨
3. 모듈 만들기:
   - `메인 캐릭터` (캐릭터 타입): 앨리스, 밥, 찰리
   - `기본 복장` (복장 타입): 교복, 드레스
   - `감정 세트` (감정 타입): happy, sad, angry, surprised
   - `아니마 품질` (품질 타입): `masterpiece, best quality` (아이템 1개)
   - `아니마 작가` (아티스트 타입): `artist:xxx, artist:yyy` (아이템 1개)
4. 배치 위자드 2단계에서:
   - 왼쪽 매트릭스: 캐릭터, 복장, 감정 모듈 추가 → 3×2×4 = 24 조합
   - 오른쪽 슬롯 매핑:
     - **Anima 긍정**: 고정=[아니마 품질, 아니마 작가] / 조합=[☑캐릭터 ☑복장 ☑감정]
     - **Anima 부정**: 고정값 `worst quality, low quality`
     - **ILXL 긍정**: 고정=[아니마 작가] / 조합=[☑캐릭터 ☑복장]
     - **ILXL 부정**: 고정값 `worst quality`
5. 조합당 50장 → 총 24 × 50 = **1,200장** 자동 생성
6. 시작 → 갤러리에서 결과 확인

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프레임워크 | Electron 39 + electron-vite |
| 프론트엔드 | Vue 3, Pinia, Vue Router, Vue I18n |
| UI 라이브러리 | Naive UI |
| 터미널 | xterm.js, node-pty |
| MCP 서버 | @modelcontextprotocol/sdk (Streamable HTTP) |
| 언어 | TypeScript 5.9 |
| 데이터베이스 | sql.js (in-memory SQLite, WASM) |
| HTTP 클라이언트 | ofetch |
| WebSocket | ws |
| 빌드 | Vite 7, electron-builder |
| 테스트 | Vitest, sql.js (in-memory test DB) |
| 코드 품질 | ESLint (flat config), Prettier |

## 프로젝트 구조

```
src/
├── main/                          # Electron 메인 프로세스
│   ├── index.ts                   # 앱 진입점
│   ├── ipc/                       # IPC 통신
│   │   ├── channels.ts            # 채널 상수
│   │   └── handlers.ts            # 핸들러 등록
│   └── services/
│       ├── database/              # sql.js DB 레이어
│       │   ├── index.ts           # 초기화, 스키마, 영속화
│       │   └── repositories/      # 리포지토리 패턴 (CRUD)
│       ├── comfyui/               # ComfyUI 서비스
│       │   ├── client.ts          # REST API 클라이언트
│       │   ├── websocket.ts       # WebSocket 연결
│       │   ├── manager.ts         # 싱글턴 매니저
│       │   ├── workflow-parser.ts  # 워크플로우 파서
│       │   └── types.ts           # ComfyUI 타입 정의
│       ├── prompt/                # 프롬프트 엔진
│       │   └── composition-engine.ts
│       ├── batch/                 # 배치 실행
│       │   ├── task-generator.ts  # 카르테시안 곱 생성
│       │   └── queue-manager.ts   # 큐 매니저
│       ├── mcp/                   # MCP 서버
│       │   ├── index.ts           # 서버 매니저 (Streamable HTTP)
│       │   └── tools.ts           # MCP 도구 정의 (15개)
│       └── terminal/              # 내장 터미널
│           └── pty-manager.ts     # PTY 인스턴스 관리 (node-pty)
├── preload/                       # 컨텍스트 브리지
└── renderer/src/                  # Vue 3 SPA
    ├── App.vue                    # 루트 (테마, i18n, IPC 이벤트)
    ├── components/layout/         # 레이아웃
    ├── stores/                    # Pinia 스토어
    ├── views/                     # 페이지 (5+1: 워크플로우, 모듈, 작업, 갤러리, 터미널 + 설정)
    ├── types/                     # 공유 타입
    └── locales/                   # i18n (ko, en)
```
